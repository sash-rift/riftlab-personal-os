# Output Standards

This document defines what good research output looks like. It establishes standards for structure, language, citation, and presentation that all agents must follow.

---

## Theoretical Foundation

Research communication is a discipline with established principles:

1. **Audience-Centered Design** (Schriver, 1997)
   - Output serves the reader's needs
   - Structure supports comprehension
   - Complexity matches audience expertise

2. **Evidence-Based Communication** (Greenhalgh, 2019)
   - Claims supported by evidence
   - Uncertainty communicated appropriately
   - Limitations acknowledged

3. **Information Architecture** (Rosenfeld & Morville)
   - Organization supports findability
   - Hierarchy reflects importance
   - Consistency aids navigation

4. **Plain Language Principles** (Cutts, 2020)
   - Clarity over complexity
   - Active voice preferred
   - Concrete over abstract

## 2. Language Standards

How to communicate research findings clearly and accurately.

### 2.1 Confidence Language Mapping

Match language to confidence level:

| Confidence | Appropriate Language | Avoid |
|------------|---------------------|-------|
| HIGH | "Is", "shows", "indicates", "demonstrates" | "May", "might", "could" |
| MEDIUM | "Suggests", "research indicates", "evidence points to" | Definitive statements |
| LOW | "Some evidence suggests", "limited data indicates" | Strong claims |
| UNVERIFIED | "Unverified reports claim", "could not be confirmed" | Any assertion of fact |

**Examples:**

HIGH confidence:
> "The market is $2.16B (Gartner, 2024, HIGH confidence)"

MEDIUM confidence:
> "Research suggests growth rates of 15-20% annually (Forrester, 2024, methodology partially disclosed)"

LOW confidence:
> "Limited data indicates pricing typically ranges $10-50/user, though sample size was small (n=12)"

UNVERIFIED:
> "Multiple aggregator sites cite $850M market size, but no primary source with methodology could be located. This figure should not be used for decisions."

### 2.2 Hedging Standards

When and how to hedge:

**Appropriate hedging:**
- Single source findings
- MEDIUM/LOW confidence data
- Extrapolations or projections
- Interpretation beyond data

**Inappropriate hedging:**
- HIGH confidence, well-sourced data
- Verified facts
- Documented methodology
- Multiple source convergence

**Hedging language hierarchy:**

| Certainty | Language |
|-----------|----------|
| Very high | "Is", "shows", "demonstrates" |
| High | "Indicates", "reveals", "confirms" |
| Moderate | "Suggests", "points to", "implies" |
| Low | "May", "might", "could" |
| Very low | "Possibly", "some evidence suggests" |
| Speculative | "It is conceivable that", "one possibility is" |

### 2.3 Active vs. Passive Voice

**Prefer active voice for clarity:**

| Passive (Avoid) | Active (Prefer) |
|-----------------|-----------------|
| "It was found that..." | "Research shows..." |
| "The market is estimated to be..." | "Gartner estimates the market at..." |
| "Customers are reported to..." | "Survey respondents report..." |

**Use passive when agent is unknown or unimportant:**

| Appropriate Passive |
|--------------------|
| "The data was collected via survey" (methodology description) |
| "This claim could not be verified" (focus on claim, not researcher) |

### 2.4 Precision vs. Vagueness

**Be specific. Vague language loses information:**

| Vague (Avoid) | Specific (Prefer) |
|---------------|-------------------|
| "Large market" | "$2.16B market (Gartner, 2024)" |
| "High growth" | "15-20% CAGR (Forrester, 2024)" |
| "Many companies" | "78% of surveyed enterprises (n=500)" |
| "Recent study" | "September 2024 Gartner survey" |
| "Experts say" | "Gartner analyst J. Smith states" |
| "Some platforms" | "Slack, Discord, and Microsoft Teams" |
| "Various approaches" | "Three primary approaches: PLG, sales-led, and hybrid" |

### 2.5 Quantitative Preservation

**Numbers must survive from source to synthesis:**

| Stage | Example |
|-------|---------|
| Source | "78% of hotel chains have adopted AI (h2c/Ireckonu study, n=171 chains)" |
| Data point | Same, with full citation |
| Analysis | Same, with confidence level |
| Synthesis | Same, not "most hotel chains" or "majority" |

**Never generalize numbers to:**
- "Many", "most", "few"
- "Large", "small", "significant"
- "Rapidly", "slowly", "steadily"

## 3. Citation Standards

How to attribute sources consistently.

### 3.1 Inline Citation Format

For claims within text:

```
[Claim] ([Source Name], [Date], [Confidence])
```

**Examples:**
```
Market size reached $2.16B in 2024 (Gartner, September 2024, HIGH confidence).

Growth is projected at 15-20% annually (Forrester, 2024, MEDIUM confidence - methodology not fully disclosed).

Some aggregators cite $850M (various, UNVERIFIED - no primary source found).
```

### 3.2 Data Point Citation Format

For formal data point documentation:

```markdown
**Claim**: [Exact claim]
**Value**: [Number/fact]
**Source**: [Full source name]
**Author**: [Author name if available]
**Date**: [Publication date]
**URL**: [Full URL]
**Tier**: [1-7]
**Confidence**: [HIGH/MEDIUM/LOW/UNVERIFIED]
**Methodology**: [If available: sample size, method]
```

### 3.3 Vendor Claim Tagging

Sources with conflicts of interest:

```
[VENDOR CLAIM] Acme Corp reports 300% engagement improvement
(Acme Corp blog, 2024, conflict of interest: self-reported)
```

### 3.4 Multi-Source Citation

When multiple sources support a claim:

```
Market growth is projected at 15-20% annually
(Gartner 2024: 18%; Forrester 2024: 15%; IDC 2024: 20%)
```

When sources conflict:

```
Market size estimates vary significantly:
- $2.16B (Gartner, 2024, methodology: enterprise survey n=500)
- $3.5B (IDC, 2024, methodology: includes adjacent markets)
- $1.2B (Forrester, 2023, methodology: pure-play vendors only)

Difference attributed to market definition scope.
```

## 4. Visual Standards

How to present data visually.

### 4.1 Table Standards

**Use tables for:**
- Comparisons across categories
- Data point summaries
- Cross-reference matrices
- Source assessments

**Table requirements:**
- Clear headers
- Consistent formatting
- No empty cells (use "-" or "N/A")
- Sorted logically (by importance, alphabetically, or chronologically)

**Example comparison table:**
```markdown
| Metric | Competitor A | Competitor B | Us |
|--------|--------------|--------------|--------|
| Price | $10/mo | $15/mo | TBD |
| Users | 50K | 30K | Beta |
| Features | 20 | 15 | 12 |

Sources: Official pricing pages, accessed 2024-01-27
```

### 4.2 Confidence Visualization

**Confidence levels in tables:**

| Finding | Confidence | Source Quality |
|---------|------------|----------------|
| Market size $2.16B | HIGH | Tier 2 (Gartner) |
| Growth 15-20% | MEDIUM | Tier 3 (multiple) |
| TAM $850M | UNVERIFIED | Tier 6 only |

**Legend:**
- HIGH = Multiple Tier 1-3 sources, methodology present
- MEDIUM = Single Tier 3-4 source or minor limitations
- LOW = Tier 5-6 sources or significant limitations
- UNVERIFIED = No credible source found

### 4.3 Data Hierarchy Visualization

**Cross-reference matrix:**

```markdown
| Finding | Angle 1 | Angle 2 | Angle 3 | Status |
|---------|---------|---------|---------|--------|
| Finding A | HIGH | HIGH | - | STRONG |
| Finding B | HIGH | CONFLICT | - | TENSION |
| Finding C | MEDIUM | - | - | WEAK |
```

## 5. Recommendation Standards

How to present actionable recommendations.

### 5.1 Recommendation Structure

Every recommendation must include:

| Component | Purpose | Required |
|-----------|---------|----------|
| What | Specific action | Yes |
| Why | Evidence that supports it | Yes |
| Confidence | How confident in evidence | Yes |
| Context fit | How it fits user's situation | Yes |
| How | Implementation approach | Yes |
| First step | Immediate next action | Yes |
| Success metric | How to measure results | Yes |
| Dependencies | What must be true/exist | If applicable |

### 5.2 Recommendation Language

**Strong recommendations (HIGH confidence + context fit):**
> "Do X. Evidence strongly supports this approach [evidence]. First step: [action]."

**Moderate recommendations (MEDIUM confidence or partial fit):**
> "Consider X. Research suggests [evidence]. Note: [caveat]. First step: [action]."

**Conditional recommendations (requires context we don't have):**
> "If [condition], then X would be appropriate because [evidence]. This recommendation requires confirming [context needed]."

**Avoid:**
> "You might want to think about possibly considering X as one option among many."

### 5.3 Recommendations Requiring Context

Explicitly flag when context is missing:

```markdown
## Recommendations Requiring Context

| Recommendation | Context Needed | Why We Can't Recommend Without It |
|----------------|----------------|-----------------------------------|
| Referral program | Pricing structure | Can't design incentives without knowing margins |
| Community strategy | Current audience size | Tactics differ for 100 vs 10,000 followers |
| SEO investment | Technical capabilities | Requires understanding of current platform |
```

## 6. Limitation Standards

How to communicate research limitations honestly.

### 6.1 Limitation Categories

Every synthesis must address:

| Category | What to Address |
|----------|-----------------|
| Data quality | What couldn't be verified, confidence distribution |
| Coverage | What wasn't researched, gaps |
| Methodology | How approach limited findings |
| Scope | What was explicitly outside scope |
| Generalizability | How broadly findings apply |
| Timeliness | When research conducted, staleness risk |

### 6.2 Limitation Language

**Be direct about limitations:**

| Weak (Avoid) | Strong (Prefer) |
|--------------|-----------------|
| "Some limitations exist" | "Three significant limitations affect these conclusions:" |
| "Data was somewhat limited" | "Of 15 data points, only 4 came from Tier 1-3 sources" |
| "More research might help" | "Verifying market size requires accessing Gartner's full methodology" |

### 6.3 Limitation Placement

- **Executive summary**: Mention most critical limitation
- **Methodology section**: Full methodology limitations
- **Findings**: Inline with relevant findings
- **Dedicated section**: Complete limitations list

## 7. Quality Checklists for Output

### Executive Summary Checklist

- [ ] Captures the key insight (not just topic)
- [ ] States why it matters
- [ ] Provides actionable direction
- [ ] Notes critical limitations
- [ ] Stands alone (readable without full report)
- [ ] 3-5 paragraphs maximum

### Findings Section Checklist

- [ ] Organized by theme/insight (not by source)
- [ ] Headlines are insights (not categories)
- [ ] Evidence cited with confidence levels
- [ ] Numbers preserved from sources
- [ ] Tensions surfaced (not hidden)

### Recommendations Checklist

- [ ] Every recommendation has evidence link
- [ ] Confidence level stated
- [ ] Context fit assessed
- [ ] Specific and actionable (not vague)
- [ ] First step defined
- [ ] Success metric provided
- [ ] Missing context flagged

### Limitations Checklist

- [ ] Data quality limitations stated
- [ ] Coverage gaps documented
- [ ] Methodology limitations noted
- [ ] Scope boundaries clear
- [ ] Generalizability addressed

### Citation Checklist

- [ ] All claims have sources
- [ ] Sources have dates
- [ ] Confidence levels assigned
- [ ] Vendor claims tagged
- [ ] Methodology noted where relevant
- [ ] Audit trail complete

## 8. Anti-Patterns

What to avoid in research output.

### 8.1 Structural Anti-Patterns

| Anti-Pattern | Problem | Fix |
|--------------|---------|-----|
| Aggregation not synthesis | "Source A said X, Source B said Y" | Synthesize into themes and insights |
| Buried lead | Key insight on page 10 | Lead with most important finding |
| Wall of text | No structure, hard to scan | Use headers, bullets, tables |
| Kitchen sink | Everything included, nothing prioritized | Prioritize, move details to appendix |

### 8.2 Language Anti-Patterns

| Anti-Pattern | Example | Fix |
|--------------|---------|-----|
| Vague quantifiers | "Many companies..." | "78% of surveyed companies (n=500)..." |
| Confidence mismatch | "The market is $850M" (UNVERIFIED data) | "Unverified reports cite $850M..." |
| Passive evasion | "It is believed that..." | "Gartner reports that..." |
| Hedge stacking | "It may possibly perhaps..." | Choose appropriate certainty level |

### 8.3 Recommendation Anti-Patterns

| Anti-Pattern | Example | Fix |
|--------------|---------|-----|
| Soft recommendations | "Consider exploring..." | "Do X. First step: Y." |
| Ungrounded recommendations | "Build a community" (no evidence) | Tie to evidence: "Build community because [evidence]" |
| Context-blind recommendations | "Launch referral program" (unknown pricing) | Flag: "Requires knowing pricing structure" |
| Vague actions | "Improve marketing" | "Run 3 A/B tests on landing page headlines" |

### 8.4 Citation Anti-Patterns

| Anti-Pattern | Example | Fix |
|--------------|---------|-----|
| Missing attribution | "Studies show..." | Name the study |
| Missing dates | "According to Gartner..." | "According to Gartner (September 2024)..." |
| Missing confidence | "$2.16B market" | "$2.16B market (HIGH confidence)" |
| Echo as validation | Same number, multiple aggregators | Note: "Same figure across sources, no primary source found" |
